Lab 2.10

Encoding Specificity / Recognition Failure

Analysis Instructions for Instructor to prepare data files

 

Instructor’s notes:

 

1)  Provide the following tables for students to plot on the graphs in the workbook:

 

  • Table 10:  percent correct, recognition and recall.

 

  • Table 20:  percent intrusions, recognition and recall

 

After running both these tables in mellab (“sanalyze”), rename the file “encode.out” to “lab2_10-tables.txt” and print it to give to students, after editing it to clean it up and make clear to students which condition is which (1=recognition, 2=recall)

 

2) Create data file for students to use.

  • Output data specifications form 10:  all variables and all trials data.  Function-3 to print it out to “encode.out”.  Then rename to to “encode-data.txt”.
  • Reboot to linux, then run “get-lab2_10-files” to create data files for students to analyze
  • Edit the resulting data file “encode.dat” and then rename it to lab2_10-data.txt, or simply rename it and leave the duplicate subjects there in order to have students think about what to do about them.  Note that the data in the table is the mean of each subject (averaging across multiple attempts at the experiment). 

 

3)  Create SPSS data files for students to analyze, lab2_10.sav, and post to web page (or have them import the data themselves to learn how to do it).

 

 

4)  Place the files lab2_10-data.txt, lab2_10-tables.txt, and lab2_10.sav, if it exists, in the course web “labs” folder.
